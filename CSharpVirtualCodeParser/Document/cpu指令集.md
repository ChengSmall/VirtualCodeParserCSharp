# 虚拟CPU指令集

## 执行过程

1. 读取指令1字节：判断指令类型

2. 根据指令类型再读取指定字节量到指令缓存

3. 将命令指针推进读取的字节数

4. 执行缓存内的指令

## 指令列表

### 指令类型

- `0x0` 空指令，指针推进到下一位，不做任何动作

- `0x1` 启用运算单元计算

    - 取后4bit判断具体运算类型：

        `0x0`：进行4字节整形运算

        `0x1`：进行8字节整形运算

        `0x2`：进行4字节无符号整形运算

        `0x3`：进行8字节无符号整形运算

        `0x4`：进行单浮点运算

        `0x5`：进行双浮点运算

    后1byte表示运算类型的值

    后2byte表示reg索引，为第一个运算值

    后3byte表示reg索引，为第二个运算值

    后4byte表示reg索引，储存返回结果

    返回结果总是会最后赋值到寄存器

- `0x2` 执行赋值操作指令

    - 取后4bit判断具体赋值类型：

        - `0x0` reg to reg

            - 从寄存器赋值到寄存器，内存按赋值到的reg的截断

            - 后1byte和后2byte，分别表示读取和写入的寄存器编号

        - `0x1` stack to reg

            - 从栈区内存赋值到寄存器，内存长度按寄存器大小
            - 后1byte表示赋值到的reg索引
            - 后2byte表示读取栈区地址值的reg索引

        - `0x2` const to reg

            - 从代码常量赋值到寄存器，常量读取长度按后2bit
            - 后1byte前6bit表示赋值到的r索引；
            - 后1byte后2bit表示代码常量读取的长度：
                - `0b00` = 1byte；`0b01` = 2byte；`0b10` = 4byte；`0b11` = 8byte

        - `0x3` reg to stack

            - 从寄存器赋值到栈区内存，内存长度按寄存器大小
            - 后1byte的前6bit表示读取的r索引
            - 后2byte表示赋值到的，包含栈区内存地址的reg索引

        - `0x4` stackSize to reg

            - 将[栈区最大字节数]值赋值到指定寄存器，内存按寄存器截断
            - 后1byte的前6bit表示赋值到的r索引

        - `0x5` stackPosMem to reg

            - 将栈区指定偏移内存写入到寄存器，内存按寄存器向后截断
            - 后1byte字节的前6bit表示写入到的reg索引；
            - 后2byte为reg索引
            - `pos = ([栈区位指针] - value);` value表示reg的值；
            - 从值[pos]处的栈区位向后读取的内存

        - `0x6` reg to stackPosMem

            - 将寄存器写入到栈区指定偏移内存，内存按寄存器向后截断
            - 后1byte读取的reg索引；
            - 后2byte为reg索引，
            - `pos = ([栈区位指针] - value);` value表示reg的值；
            - 从[pos]处的栈区位向后写入的内存

        - `0x7` Memory to reg

            - 从父沙箱内存赋值到寄存器，内存长度按寄存器大小
            - 后1byte为写入到的reg索引，后2byte表示reg索引，为父沙箱内存地址值
            - 内存读取长度按赋值寄存器大小

        - `0x8` reg to Memory

            - 从寄存器赋值到父沙箱内存，内存长度按寄存器大小
            - 后1byte为读取到的reg索引，后2byte表示reg索引，为要写入的父沙箱内存值
            - 获取的地址长度按进程位大小
            - 写入的内存按寄存器大小

        - `0x9` （未定义）
        - `0xA` （未定义）
        - `0xB` （未定义）
        - `0xC` （未定义）
        - `0xD` （未定义）
        - `0xE` （未定义）
        - `0xF` （未定义）

- `0x3` 启用跳转指令修改程序计数器指针

    - 取后4bit判断条件跳转类型：

        - `0x0` 无条件跳转，到寄存器地址

            - 读取1个byte位reg索引，表示跳转到的地址

        - `0x1` 无条件跳转，到常量地址

            - 读取4个byte值，表示跳转到的地址值

        - `0x2` 非0跳转，到寄存器地址

            - 后1byte值表示reg索引，为是否非0的寄存器；
            - 后2byte，表示reg索引，值为跳转到的位置；
            - 若寄存器非0，则跳转到指定位置

        - `0x3` 非0跳转，到常量地址

            - 后1byte表示reg索引，为是否非0的寄存器；
            - 后2byte到后5byte组成4字节值，表示其跳转位置：
            - 若寄存器非0，则跳转到指定位置

        - `0x4` （未定义）
        - `0x5` （未定义）
        - `0x6` （未定义）
        - `0x7` （未定义）

- `0x4` 执行栈操作指令-入栈（入栈表示为增加栈区位指针值）

    - 取后4bit判断具体入栈方式：

        - `0x0` sizeof(reg)

            - 使用寄存器容量大小表示入栈字节量
            - 后1byte：表示reg索引，值记作入栈后写入的数据
            - 入栈字节量取决于寄存器容量

        - `0x1` byte

            - 使用值量表示入栈字节量
            - 向后1byte，表示寄存器索引；
            - 寄存器的值表示入栈的大小

        - `0x2` （未定义）
        - `0x3` （未定义）

- `0x5` 执行栈操作指令-弹栈（弹栈表示为减少栈区位指针值）

    - 取后4bit判断具体弹栈方式：

        - `0x0` pop reg
            - 弹出栈区内存到寄存器
            - 后1byte表示写入到的reg索引；
            - 弹栈的大小使用寄存器大小

        - `0x1` pop size

            - 弹出栈区，舍弃值
            - 后1byte表示reg索引，其值表示要弹出的字节大小的值

        - `0x2` （未定义）
